# Metrics Interpretation Skill

Целевая аудитория — топ-менеджер без ML-бэкграунда. Голые числа для него шум. Каждое число в моём ответе должно быть **читаемым с одного взгляда**.

## Шаблон «число + расшифровка + маркер»

Минимальный формат для одной метрики:

> `focus = 0.41` → доля рабочего времени в концентрированной работе (без переключений между задачами/чатами). Норма для роли — `0.55–0.75`. У сотрудника **повышено фрагментировано**.

Структура: `имя_метрики = значение` → *что измеряется* (*шкала/единица*). *Норма/диапазон*. **Качественный маркер**.

## Качественные маркеры

- **норма** — значение в типичном диапазоне для роли/команды.
- **повышено** / **понижено** — выходит за норму, но не критично; стоит обратить внимание.
- **тревожно** — требует разговора с человеком/руководителем.

Если порога в `data_engine` нет — я явно говорю: *«моя интерпретация, а не порог системы»* (P6 Authenticity).

## Раскрытие жаргона

В первом упоминании в ответе раскрываю:

- **focus** — доля времени в концентрированной работе (без переключений).
- **meetings_per_day** — среднее число встреч в рабочий день.
- **attrition_score** — модельная вероятность ухода сотрудника в течение 6 месяцев (0–1).
- **role_success** — модельная оценка соответствия должности (0–1).
- **degree centrality** / **degree** — количество прямых рабочих связей с коллегами.
- **weight** (в графе коллабораций) — суммарная интенсивность взаимодействия (встречи, переписка, совместные задачи).

## «Как читать таблицу»

Если в ответе таблица с **≥3 числовыми столбцами** — перед ней короткий абзац (1–3 строки):

> *Сортировка по сумме весов связей. `degree` — число коллег, с кем есть рабочее взаимодействие; `weight_sum` — суммарная интенсивность контактов; чем выше — тем плотнее человек в коммуникации.*

## Чек-лист перед отправкой ответа

Прежде чем отправить ответ с метриками, я проверяю:

1. Каждое число имеет расшифровку (что измеряется)?
2. Каждое число имеет качественный маркер (норма/повышено/тревожно)?
3. Жаргон раскрыт в первом упоминании?
4. Для таблицы с ≥3 числовыми столбцами есть «как читать»?
5. В конце — сводный вывод 1–2 предложения «что это значит для сотрудника/менеджера»?

Если хотя бы один пункт «нет» — переписываю до отправки.

## Антипаттерны

Чего я **не делаю**:

- ❌ «focus=0.41, meetings=3.6» — голые числа без шкалы и интерпретации.
- ❌ «суммарный вес 6.80» — без объяснения, что такое «вес» и много это или мало.
- ❌ «degree centrality 28» — без расшифровки термина и без сравнения с нормой.
- ❌ «attrition_score=0.62, role_success=0.71» — два числа подряд без интерпретации.
- ❌ Таблица с 5 числовыми столбцами без абзаца «как читать» сверху.

## Баланс с «не-режим-отчёта»

Правило не должно превращать ответ в простыню. Маркер — это одно слово (**норма** / **повышено** / **тревожно**), а не абзац. «Как читать таблицу» — 1–3 строки. Если ответ на короткий вопрос с одной метрикой — достаточно одной фразы расшифровки. Длина не цель, ясность — цель.
